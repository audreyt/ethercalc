EtherCalc is the spreadsheet equivalent of a wiki page: no logins, no editing locks. Everyone with the link can edit the same sheet at the same time.

This guide replaces the old [GitHub wiki](https://github.com/audreyt/ethercalc/wiki), which is no longer maintained. For API and deployment topics, see [Self-hosting](/self-host/) and [API.md](https://github.com/audreyt/ethercalc/blob/main/API.md) in the repository.

## Use cases

When might you reach for EtherCalc?

- **Scheduling** — participants update their own availability; everyone sees the latest headcount per slot without e-mail ping-pong.
- **Carpooling** — who rides with whom, how many seats remain.
- **Peer grading** — students score each other's work in a shared grid.
- **Live run-of-show** — a producer updates a roadmap sheet; studio staff follow along with no refresh button.

## Getting started

1. Open your EtherCalc instance (for example `https://ethercalc.net/=_new`) to create a workbook with sheet tabs. Use `/_new` only when you specifically need a single-sheet room.
2. Share the room URL. Anyone with the link can edit unless the host set an `ETHERCALC_KEY` (see [Self-hosting](/self-host/)).
3. Type in cells as you would in any spreadsheet. Changes sync in real time.
